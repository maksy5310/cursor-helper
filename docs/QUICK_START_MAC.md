# Mac 和远程开发快速指南

## 🎯 问题现象

- ✅ Windows 下正常工作
- ❌ Mac 下看不到会话列表
- ❌ 远程开发（SSH）看不到会话列表

## 🔧 快速解决

### 步骤1：运行诊断

1. 打开命令面板：`Cmd+Shift+P` (Mac) 或 `Ctrl+Shift+P` (Windows)
2. 输入：`Cursor Assistant: 诊断工作空间路径`
3. 按回车运行

### 步骤2：查看结果

**✅ 如果显示成功：**
```
✓ 工作空间路径匹配成功
✓ 已找到对应的数据库文件
```
→ 问题已解决，会话列表应该正常显示

**⚠️ 如果显示未找到数据库：**

查看诊断输出中的"可能原因"部分，常见情况：

#### 情况1：首次打开工作空间
```
原因：首次打开，数据库尚未生成
```
**解决：**
1. 在 Cursor 中使用一次 Composer（AI 对话）
2. 等待 1-2 分钟
3. 重新运行诊断命令

#### 情况2：路径不匹配（远程开发）
```
原因：本地路径和远程路径不匹配
```
**解决：**
确保本地和远程使用相同的项目目录名称

例如：
- ✅ 远程：`/home/user/myproject` → 本地：`/Users/user/myproject`
- ❌ 远程：`/home/user/server-app` → 本地：`/Users/user/client-app`

### 步骤3：查看详细信息

点击诊断结果中的 **"查看输出"** 按钮，可以看到：
- 所有工作空间的路径信息
- 路径匹配的详细过程
- 具体的匹配或不匹配原因

## ❓ 常见问题

**Q: 诊断显示成功，但还是看不到会话列表？**

A: 
1. 重启 Cursor
2. 确认是否使用过 Composer（只有 Composer 的对话会显示）
3. 查看输出面板是否有其他错误

**Q: 远程开发总是匹配失败？**

A:
1. 检查项目名称是否一致
2. 在本地使用与远程相同的项目名称
3. 或者在远程创建与本地相同的路径结构

**Q: 可以手动指定数据库路径吗？**

A: 当前版本不支持，但已在计划中。请关注后续更新。

## 💡 提示

- 路径匹配使用智能算法，会自动处理大部分情况
- 如果诊断显示"后缀匹配成功"，说明远程路径匹配功能正常工作
- Mac 用户请确保路径以 `/Users/` 开头（标准 macOS 用户目录）
